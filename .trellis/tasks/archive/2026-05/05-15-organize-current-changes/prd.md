# 整理并提交当前变动

## Goal

研究当前工作树中的未提交变动，判断每组变动的来源、风险和提交边界，然后在不要求用户中途决策的前提下完成必要整理、验证和提交。

## Requirements

- 保留并提交有价值的项目变更、Trellis 模板同步和项目规范更新。
- 不把 Trellis update 产生的临时对照文件或备份目录纳入提交。
- 对当前 diff 做足够证据检查，包括 Trellis dry-run、Git 状态、CI/脚本/Gradle 配置上下文、项目规范和近期提交历史。
- 选择合理提交边界，避免把 Trellis 模板同步、发布打包修正、Gradle 依赖解析策略混成一个难以审查的提交。
- 提交前运行与改动相关的最小验证；若某项验证受环境限制无法完成，记录具体原因。
- 完成后归档当前 Trellis task，并在最终回复中给出提交列表、验证结果和残留风险。

## Acceptance Criteria

- [x] 当前未提交变动被分类，并有明确保留/排除理由。
- [x] Trellis task 工件记录最终提交策略和验证策略。
- [x] 需要保留的变动被拆成语义清晰的 Git commit。
- [x] 临时忽略文件不进入提交；若保留在工作区，最终说明其状态。
- [x] 运行并报告最小验证结果。
- [x] 当前 Trellis task 被完成或归档，工作树最终状态可解释。

## Notes

- 用户明确要求“充分研究后行动，中途不要让用户来决定如何做”，因此本任务中可根据证据自行选择提交边界。
- 当前工作树包含 Trellis 更新、发布打包/CI、Android Gradle 仓库镜像、脚本修正和项目规范文件。
